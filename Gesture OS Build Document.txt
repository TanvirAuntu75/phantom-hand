01 // PROJECT IDENTITY
Codename: PHANTOM HAND
Classification: Personal Vision Project — Keyboard-Free Gesture OS
▸ Vision Statement
A zero-keyboard, zero-mouse operating interface where every action — drawing, navigating, executing commands, interacting with AI — is performed through precise hand gestures captured by a standard webcam.
Think less 'art app' and more 'operating system layer' that sits on top of your screen.
The aesthetic is cold, clinical, and inhuman — holographic HUD elements, real-time data streams, glowing overlays — the kind of interface that makes people think a machine is watching them, not helping them.
▸ Core Philosophy
No keyboard. No mouse. No touch. Hands only.
No generic UI. Every pixel is custom-designed.
No limitations. Shape recognition, letter recognition, AI completion, voice fallback.
No cost. 100% free stack. Antigravity + Gemini tokens as the only 'paid' resource.
No imitation. No template cloning. Every feature engineered from scratch.
▸ Inspiration Architecture
JARVIS: Proactive AI overlay, voice-command fallback, real-time data display
FRIDAY: Gesture-driven task execution, contextual awareness, command routing
EDITH: Tactical interface mode, target locking, precision gesture mapping
Minority Report: Physical gesture as primary navigation paradigm
Unique Layer: Air-written text converted to real OS commands via Gemini interpretation

| PHASE 1 | CORE HAND TRACKING ENGINE | TIMELINE: Day 3–5 |  | :--- | :--- | :--- | | Focus | The foundation everything else is built on. Must achieve 30+ FPS before advancing. | |
◆ MediaPipe Pipeline
Initialize MediaPipe Hands with model_complexity=1 (full, not lite)
Set max_num_hands=2 — always track both hands
Set min_detection_confidence=0.75 and min_tracking_confidence=0.75
Capture BGR frame from webcam, flip horizontally for mirror effect
Convert BGR to RGB before passing to MediaPipe
Extract all 21 landmark (x, y, z) tuples per detected hand
Extract handedness classification — Left or Right label
Extract per-hand confidence score
◆ Kalman Filter Smoother
Implement Kalman filter for each of the 21 landmark points independently
State vector: position (x, y, z) + velocity (vx, vy, vz)
Tune process noise and measurement noise matrices
Result: sub-5px jitter on stable held positions
Implement velocity threshold — only activate drawing when velocity is deliberate
◆ Performance Optimization
Target: 30 FPS minimum on standard laptop webcam (no GPU required)
Run MediaPipe processing on separate thread with queue
Reduce tracking resolution to 640x480, display at 1280x720
Profile and eliminate all blocking calls in main loop
Implement FPS counter — display in HUD at all times
◆ Hand Data Output Schema
Each frame outputs: timestamp, hand_count, [hand_id, handedness, confidence, landmarks[21]]
Landmarks normalized 0.0-1.0 on both axes
Z-axis: 0 = wrist plane, negative = closer to camera
Output continuously to gesture engine via internal queue
| PHASE 2 | GESTURE RECOGNITION SYSTEM | TIMELINE: Day 6–10 |  | :--- | :--- | :--- | | Focus | All 30 gestures. No approximation. Each gesture triggers one and only one action. | |
◆ Rule-Based Gesture Layer (First Pass)
Build finger state detector — for each finger: extended or curled
Thumb uses X-axis comparison (tip vs IP joint)
Index through Pinky use Y-axis comparison (tip vs PIP joint)
Compute Euclidean distance between all relevant tip pairs
Compute wrist-to-fingertip vectors for direction analysis
Compute palm normal vector for palm facing direction
Build gesture state machine — debounce each gesture for 3 consecutive frames
Implement gesture cooldown — prevent accidental rapid retriggering
◆ Machine Learning Gesture Layer (Second Pass)
Collect training data — 150 samples per gesture x 30 gestures = 4,500 samples
Record in-app using a data collection mode (gesture hold = record)
Feature engineering — flatten 21 landmarks to 63 floats, normalize by palm size
Train TensorFlow Lite classification model — 3-layer MLP
Target accuracy: 97%+ on validation set
Export to .tflite and deploy in real-time pipeline
Rule-based and ML outputs combined via confidence voting
◆ Gesture State Machine
States: IDLE → DETECTING → CONFIRMED → ACTIVE → RELEASED
Transitions require N consecutive frames in same gesture class
Hold-triggered gestures require 30-frame hold (1 second at 30fps)
Swipe gestures require velocity vector in correct direction
Two-handed gestures only trigger if both hands are in frame
Log all gesture events with timestamp to gesture_log.json
◆ Custom Gesture Trainer
Build in-app UI mode: record your own gesture
User holds gesture for 3 seconds — 90 frames captured automatically
System trains mini-model on new gesture + existing library
Export custom gesture pack as JSON — shareable with others

◆ Canvas Architecture
Infinite canvas with 4K backing store (3840x2160 NumPy array)
Viewport: 1920x1080 window into canvas, pan/zoom adjustable

Layer compositing on render — flatten only for display, not storage

◆ Motion Smoothing
Gaussian-weighted moving average over last 8 tracked points
Bezier curve interpolation between spaced points
Velocity-adaptive smoothing — faster gesture = more smoothing
Minimum inter-point distance threshold — avoid point clouds on slow strokes
◆ All 12 Brush Modes — Implementation Details
PENCIL — OpenCV polylines, width 1-3px, no anti-aliasing, raw edge
BRUSH — Gaussian-blurred circles at each point, tapered using velocity
AIRBRUSH — Random scatter of N dots within radius R of finger, density = pressure
CALLIGRAPHY — Angle-sensitive width modulation via wrist tilt
NEON GLOW — Three layered circles: core white, mid color, outer bloom with alpha blend
LASER — Bright center with rapid fade trail, sci-fi aesthetic
PARTICLE PEN — Each tracked point spawns 8 particles with random velocity + gravity
MIRROR — Duplicate every point across one or both symmetry axes in real-time
ERASER — Circular collision on all stroke segments in bounding box, remove intersects
CLONE STAMP — Sample 64x64 region on pinch, paste at current position on draw
PATTERN FILL — Flood fill from closed shape with selected texture tile
3D RIBBON — Stroke has visible depth based on Z-depth of hand
◆ Real-Time Rendering Pipeline
Draw to offscreen buffer, composite with camera frame via alpha blend
Camera frame opacity: 60% — drawing opacity: 100%
Target render latency: under 16ms (1 frame at 60fps)
JPEG encode at 85% quality for WebSocket transmission
Only transmit delta regions when possible to reduce bandwidth
| PHASE 4 | SHAPE RECOGNITION SYSTEM | TIMELINE: Day 17–21 |  | :--- | :--- | :--- | | Focus | 25 shapes. Detect, snap, and render clean geometry from freehand gestures. | |
◆ Detection Pipeline
Stroke is analyzed on fist-close gesture (end of stroke)
Normalize stroke points to 0-1 coordinate space
Resample stroke to uniform 128 points for consistent analysis
Compute bounding box, centroid, area, perimeter
Compute curvature profile along stroke length
Compute convex hull and convexity defects
Compute vertex candidates using Douglas-Peucker simplification
◆ Shape Classification Logic
CLOSED SHAPE DETECTION — start and end point within 10% of bounding box diagonal
CIRCLE — Radius coefficient of variation < 12% across all centroid distances
ELLIPSE — Fit ellipse via OpenCV fitEllipse, check residual error
SQUARE — 4 corners, equal sides within 5%, all angles 88-92 degrees
RECTANGLE — 4 corners, two pairs of equal sides, all angles 88-92 degrees
TRIANGLE (all types) — 3 corners, classify by side ratios and angles
DIAMOND — 4 equal sides, vertex at topmost point, angles not 90 degrees
PARALLELOGRAM — 2 pairs parallel sides, no right angles
TRAPEZOID — 1 pair parallel, 1 pair divergent sides
PENTAGON — 5-vertex polygon fit, equal angle check
HEXAGON — 6-vertex polygon fit, regular spacing check
OCTAGON — 8-vertex polygon fit, near-circular bounding
STAR (5pt) — 10-vertex alternating pattern, concave defect detection
STAR (6pt) — 12-vertex dual-triangle interference pattern
CROSS — 4 arms extending from center, orthogonal vectors
SPIRAL — Continuously expanding radius over stroke progression
ARC — Partial circle fit, angular coverage 30-300 degrees
CRESCENT — Two arcs, inner arc subtract from outer arc
HEART — Dual bezier arcs converging at bottom point
ARROW (single) — Line + triangular cap, direction vector extracted
ARROW (double) — Two caps on single shaft
S-CURVE — Single inflection point detected in curvature profile
LINE — Maximum deviation from best-fit line < 5px
FREEFORM — Fallback, store as smooth Bezier approximation
