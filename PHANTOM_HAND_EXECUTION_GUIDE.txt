Step 1 — Gesture Engine Core (Antigravity)
PROMPT — ANTIGRAVITY BACKEND AGENT — gesture_engine.py (Part 1 of 2)
Build backend/core/gesture_engine.py for the Phantom Hand system.
This is Part 1: the rule-based detection layer.
CONTEXT: The hand_tracker.py module outputs 21 landmarks per hand as
normalized (x, y, z) tuples. This module takes those landmarks and
determines what gesture the user is making.
PART 1 — RULE-BASED DETECTION:
1. Build a FingerStateDetector class:
   - Method: get_finger_states(landmarks) -> dict
   - Returns: {thumb: bool, index: bool, middle: bool, ring: bool, pinky: bool}
   - True = finger is extended, False = finger is curled
   - Thumb: compare tip (landmark 4) X position vs IP joint (landmark 3) X
   - Other fingers: compare tip Y vs PIP joint Y (tip higher = extended)
   - Landmark indices: thumb_tip=4, index_tip=8, middle_tip=12, ring_tip=16, pinky_tip=20
   - Landmark PIP indices: index_pip=6, middle_pip=10, ring_pip=14, pinky_pip=18
2. Build a GestureDetector class:
   - Uses FingerStateDetector plus distance and angle calculations
   - Must detect these gestures by name:
   DRAW: only index extended, others curled, palm facing camera
   PRECISION_LINE: index + middle extended, others curled
   STAMP: thumb + index pinch (distance between tips < 0.04)
   COLOR_PICK: thumb + index + middle pinch (all three tips within 0.05 of each other)
   STOP: full fist (all fingers curled), held for 1 second = 30 frames
   ERASE_H: peace sign (index+middle up) with horizontal wrist velocity
   ERASE_V: peace sign (index+middle up) with vertical wrist velocity
   CLEAR_CANVAS: all 5 fingers extended (open palm)
   CONFIRM: index finger tap (rapid down-up velocity spike on index tip)
   SHAPE_LOCK: index tip traces a full circle (closed path)
   UNDO: thumbs up (thumb up, others curled)
   REDO: thumbs down (thumb down, others curled, hand inverted)
   SAVE: open palm pushing forward (all extended, Z-velocity negative)
   COLOR_NEXT: index pointing right + rightward velocity
   COLOR_PREV: index pointing left + leftward velocity
   BRUSH_UP: index pointing up + upward velocity
   BRUSH_DOWN: index pointing down + downward velocity
   ROTATE_CW: wrist rotate clockwise (angular velocity of palm normal)
   ROTATE_CCW: wrist rotate counter-clockwise
   ZOOM_IN: two hands detected, distance between wrists increasing
   ZOOM_OUT: two hands detected, distance between wrists decreasing
   AI_MODE: index + pinky extended (horns shape), others curled
   LETTER_MODE: L-shape = thumb extended right + index extended up
   VIEW_3D: three fingers (index+middle+ring) swipe upward
   VIEW_2D: three fingers swipe downward
   EXPORT: four fingers (index+middle+ring+pinky) tap together
   VOICE: only pinky extended, others curled
   COLLAB: two hands, both index fingers pointing toward each other
   LAYER_CYCLE: wrist tilt side-to-side (roll angle change > 20 degrees)
   AIR_WRITE: index extended moving slowly (velocity < 0.01/frame)
3. Gesture State Machine:
   - States per gesture: IDLE, DETECTING, CONFIRMED, ACTIVE, RELEASED
   - A gesture must be detected for 3 consecutive frames to CONFIRM
   - Hold gestures (STOP, AI_MODE) require 30 frames to ACTIVATE
   - After a gesture RELEASES, 10-frame cooldown before same gesture can trigger again
   - Two-handed gestures only trigger if both hands are in frame
4. Output from GestureDetector.process(frame_data) -> dict:
   {
     gesture_name: str or None,
     confidence: float 0.0-1.0,
     state: str (IDLE/DETECTING/CONFIRMED/ACTIVE/RELEASED),
     hand_count: int,
     active_hand: dict (dominant hand landmarks)
   }
5. Logging: write every CONFIRMED gesture event to gesture_log.json
   with timestamp, gesture name, confidence, duration.
Use Python type hints and docstrings on everything.
No external ML libraries in this file — pure geometry and physics.

Step 2 — Motion Smoother
PROMPT — ANTIGRAVITY BACKEND AGENT — motion_smoother.py
Build backend/core/motion_smoother.py.
This module smooths the path of the drawing fingertip before it
is sent to the drawing engine. Without this, strokes look jagged.
1. Class: MotionSmoother
   Method: smooth_point(point: tuple) -> tuple:
   - Accepts (x, y, z) normalized coordinates
   - Maintains internal history of last 8 points
   - Returns Gaussian-weighted average of history
   - Weights: more recent points weighted higher
   - Must operate in under 1ms
   Method: get_bezier_path(points: list) -> list:
   - Takes list of raw points
   - Returns much larger list of interpolated points along Bezier curves
   - Result: smooth curves instead of jagged lines between samples
   Method: get_velocity(recent_points: list) -> float:
   - Returns magnitude of movement speed based on last 5 points
   - Used to determine if user is drawing (deliberate) or hovering (idle)
   Property: is_deliberate_motion -> bool:
   - Returns True if current velocity is above threshold 0.003
   - Drawing only happens when this is True
   Method: reset():
   - Clears history (call when hand leaves frame or gesture changes)
2. Class: VelocityAdaptiveSmoother(MotionSmoother):
   - Overrides smooth_point to use MORE smoothing when moving fast
   - Uses LESS smoothing when moving slow (for precision detail work)
   - This feels like how real pens behave
Use only numpy. Type hints and docstrings.
Step 3 — Brush System
PROMPT — ANTIGRAVITY BACKEND AGENT — brush_system.py
Build backend/core/brush_system.py for Phantom Hand.
This module implements all 12 brush modes. Each mode takes a stroke
object and renders it onto a Layer's numpy array.
For each brush, build a separate class inheriting from BaseBrush.
BaseBrush has one abstract method: render(stroke, layer_array) -> None
Implement all 12 brush classes:
1. PencilBrush:
   - cv2.polylines on layer array
   - Width: 1-3px, antialiasing OFF
   - Hard, raw edge — exactly like HB pencil
2. BrushBrush:
   - Gaussian-blurred filled circles at each point
   - Width tapers at stroke start and end using velocity
   - Soft edges via alpha blending
   - Z-depth modulates width: closer hand = wider stroke
3. AirbrushBrush:
   - At each point, scatter N random dots within radius R
   - N and R increase with Z-depth (pressure)
   - Each dot has random alpha 30-80%
   - Result looks like aerosol spray
4. CalligraphyBrush:
   - At each point draw an ellipse
   - Ellipse major axis angle = wrist tilt angle from hand data
   - Width modulates with stroke speed
   - Classic calligraphy thick/thin variation
5. NeonGlowBrush:
   - Draw three concentric blurred circles at each point:
     Inner: white, 3px, 100% alpha
     Middle: chosen color, 8px, 80% alpha
     Outer: chosen color, 20px, 30% alpha (bloom)
   - Additive blending (add pixel values, clamp at 255)
   - Result: glowing neon tube
6. LaserBrush:
   - Bright white-hot center (2px, 100%)
   - Color trail that fades over last 5 recorded points
   - Chromatic aberration: R channel offset +2px, B channel -2px
   - Sci-fi energy beam aesthetic
7. ParticlePenBrush:
   - At each tracked point, spawn 8 child particles
   - Each particle has: random velocity, gravity (-0.001 per frame), lifespan 20 frames
   - Particles rendered as 1-2px dots
   - Maintain particle simulation state between render calls
   - Result: drawing leaves sparkle trails
8. MirrorBrush:
   - Wrapper that passes each point through another brush
   - Also sends the mirrored point (1-x, y) to same brush
   - Optionally also mirrors vertically (x, 1-y)
   - Property: axis — 'vertical', 'horizontal', 'both'
9. EraserBrush:
   - For each point, set pixels within radius R to transparent
   - Use cv2.circle with alpha=0
   - Width controlled by brush size setting
10. CloneStampBrush:
    - On STAMP gesture: sample 128x128 region around finger position
    - On DRAW: paste sampled region centered on current finger position
    - Show ghost outline of sample region as feedback
11. PatternFillBrush:
    - On closed stroke: detect enclosed region using cv2.findContours
    - Tile selected texture image inside region
    - Available textures: solid, dots, crosshatch, honeycomb, circuit
12. Ribbon3DBrush:
    - Width = base_width * (1 + stroke.z_depth)
    - Draw a trapezoid between consecutive points (not a line)
    - Left edge is lighter, right edge is darker (simulated lighting)
    - Creates volumetric tape/ribbon appearance
BrushRegistry:
- Dict mapping brush mode name to brush class instance
- Factory method: get_brush(mode_name) -> BaseBrush
All classes must have type hints and docstrings.
All render methods must complete in under 5ms for 1920x1080 canvas.
CHECK before Phase 4: The drawing system needs a quick test. Ask Antigravity to write a test script that simulates 5 brush strokes on a test canvas for each brush mode and saves the result as test_brushes.png. Open that image. All 12 brushes must look visually distinct from each other.

PROMPT — JULES — shape_recognizer.py
Build backend/core/shape_recognizer.py for the Phantom Hand system.
PURPOSE: When a user finishes drawing a stroke (detected by STOP gesture),
this module analyzes the stroke and determines what shape it is.
If it recognizes a known shape, it returns a clean fitted version.
INPUT: A list of (x, y) coordinate tuples representing a stroke path.
OUTPUT: A dict: { shape_name: str, fitted_points: list, confidence: float }
PROCESSING PIPELINE:
1. Normalize points to 0-1 space
2. Resample to exactly 128 evenly-spaced points
3. Compute: bounding box, centroid, area, perimeter
4. Compute curvature at each point
5. Compute convex hull and convexity defects
6. Simplify with Douglas-Peucker to find vertices
7. Detect if shape is closed (start/end within 8% of bounding diagonal)
8. Run shape classifiers in order of specificity
SHAPE CLASSIFIERS — implement each as a private method returning (bool, float):
The float is confidence 0.0-1.0.
Closed shapes:
_is_circle(points) — radius variance < 12% of mean radius
_is_ellipse(points) — fitEllipse residual < threshold, not circle
